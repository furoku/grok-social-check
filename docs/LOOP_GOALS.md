# Grok Social Check — ループ構築目標

この文書は **構築ループ** の北極星・停止条件・微分目標・ズレ検知を定義する。  
各イテレーションは「計画 → 実装 → 検証 → 振り返り」の順で回し、ズレがあれば次の微分目標を修正してから続行する。

---

## 1. 最終ゴール（North Star）

**ユーザーが PC の Chrome で X / Threads の投稿を開き、「Grokでチェック」から参考分析（ファクト整理 + 左右寄り参考ラベル）を安定して得られる。**

| 観点 | 完了の定義 |
|------|------------|
| 機能 | 投稿本文取得 → xAI API → 日本語 JSON → パネル表示が一連で動く |
| 対象 | X タイムライン + Threads タイムライン（個別 URL は v1 任意） |
| 品質 | 誤抽出・空投稿・API エラー時にユーザーが次の行動を理解できるメッセージ |
| 安全 | API キーはオプション保存のみ。リポ・ログ・Discord にキーを出さない |
| 検証 | `npm run check` が常に緑。PC では `npm run check:pc` も緑 |
| 説明責任 | UI に「公式ファクトチェックではない」が常に見える |

---

## 2. 停止条件

### 2.1 成功停止（ループ終了してリリース候補へ）

次を **すべて** 満たしたら構築ループを止め、CWS 検討 or 限定配布フェーズへ移る。

1. **G0** 最終ゴール表の「完了の定義」を PO が口頭または Discord で OK  
2. **G1** Mac/Windows の Google Chrome で `npm run check:pc` が 2 回連続成功  
3. **G2** 手動シナリオ 3 本（下記）がスクショ or 短文ログ付きで記録済み  
4. **G3** 既知の P0/P1 バグが 0（`docs/LOOP_GOALS.md` のバックログ参照）  
5. **G4** manifest の権限が `storage` + `api.x.ai` のみ（意図しない権限増なし）

### 2.2 一時停止（ループは続けるが実装を止める）

- x.com / Threads の DOM 破壊で `smoke:chrome` が 3 回連続失敗 → **抽出・セレクタ修正イテレーション** に専念  
- xAI API のモデル名廃止・レート制限で本番が使えない → **設定・フォールバック** のみ触る  
- PO が「左右判定の文言を変えたい」等の仕様変更 → 微分目標を差し替えてから再開  

### 2.3 中止（スコープ縮小）

- CWS 方針で `host_permissions` 追加が許されない → **Worker プロキシ版** に切替（別プロジェクト）  
- 公式 API 利用不可が継続 → 拡張は「下書きコピー + 手動 Grok」に縮小

## 3. 構築ループ（1 イテレーション = 1〜3 日想定）

```
┌─────────────┐
│ 微分目標 Dx │  ← このイテレで増やす能力だけを書く
└──────┬──────┘
       ▼
┌─────────────┐
│ 実装 + 差分 │  小さな PR / コミット
└──────┬──────┘
       ▼
┌─────────────┐
│ 検証ゲート   │  smoke → test → (PC) smoke:chrome → 手動 1 本
└──────┬──────┘
       ▼
┌─────────────┐
│ ズレ判定     │  下記シグナルと照合
└──────┬──────┘
       ├─ ズレ小 → 次の D(x+1) へ
       └─ ズレ大 → 微分目標を書き直し、同じ Dx を再実行
```

**ループ内の必須コマンド（毎イテレ終わり）**

```bash
npm run check
# PC 作業マシンでは追加
npm run smoke:chrome
```

## 4. 微分目標（ロードマップ）

能力を **縦に積む** 順序。前の段が緑でないと次に進まない。

| ID | 微分目標 | 完了条件（客観） | 状態 |
|----|----------|------------------|------|
| D0 | リポ・MV3 骨格・静的 smoke | `npm run smoke` 緑、GitHub push 済 | ✅ |
| D1 | 投稿抽出の単体テスト | `npm run test` 6/6 緑、X/Threads fixture | ✅ |
| D2 | PC Chrome で拡張ロード | `smoke:chrome` が options + x.com マーカーまで | 🔄 PO PC 未確認 |
| D3 | 実 API で 1 投稿分析 | オプションにキー、ボタン 1 回でパネルに JSON 要約 | ⬜ |
| D4 | Threads 実 DOM でボタン表示 | threads.com で `.grok-check-btn` 可視（smoke または手動） | ⬜ |
| D5 | エラー UX・レート抑止 | キー未設定・429・空本文で日本語メッセージ固定 | ⬜ |
| D6 | プロンプト品質 v2 | 同一投稿で fact / lean の再現性を PO が「参考になる」と判断 | ⬜ |
| D7 | パッケージ・簡易プライバシー文 | README + PRIVACY.md、ZIP 手順 | ⬜ |

**次に取るべき微分目標:** **D2 → D3**（PC で Chrome smoke を通し、実キーで 1 本成功）

## 5. ズレ検知（Drift Signals）

| シグナル | 意味 | 対処（微分目標の修正） |
|----------|------|------------------------|
| `npm run check` が赤 | 基盤退行 | 新機能を止め、直前コミットと diff。D0/D1 に戻る |
| unit は緑だが smoke:chrome が赤 | 実ブラウザ / DOM / manifest | D2 または D4 に固定。セレクタ・matches を直す |
| ボタンは出るが本文が空 | 抽出ロジックズレ | `lib/post-extract.js` + fixture 追加。D1 拡張 |
| API は 200 だが JSON パース失敗 | プロンプト / モデル | `lib/prompt.js` とモデル名。D6 |
| PO「左右が毎回違う」 | 仕様どおり（参考ラベル）か品質問題か | 文言で不確実性を強調 or temperature 下げ。D6 |
| キーが漏れた疑い | プロセス破綻 | 即ローテーション。コードにキー直書き禁止を再確認 |
| 権限が manifest に増えた | セキュリティ・CWS リスク | 意図しない diff を revert。§2.1 G4 |

**週次（または 3 イテレごと）のズレレビュー（5 分）**

1. 上表で 🔴 になっている行はあるか  
2. バックログの P0/P1 が増えていないか  
3. 最終ゴールの表と PO の期待がまだ一致しているか

## 9. このループの使い方（PO / エージェント）

1. 作業開始時: **§4 で ⬜ のうち最上段** を今回の微分目標にする  
2. 作業終了時: `npm run check` 結果と §5 シグナルを 3 行で `progress-log` に追記  
3. 微分目標完了時: 表の状態を ✅ に更新して commit  
4. §2.1 を満たしたらループ停止し、配布判断へ  

**現在の推奨アクション:** PO の PC で `npm run check:pc` → 成功したら **D3**（実キー 1 投稿）を 1 イテレで完了させる。
